# Initial `.bac` System Plan

## 背景

本项目要开发一个可被 AI 编程工具调用的贡献归因 tool。该 tool 负责创建、追加和验证 `.bac` 文件，用于区分具体项目中人类、AI、工具和系统事件的贡献。

## 初始目标

- 定义 `.bac` 文件格式和版本字段
- 定义贡献事件模型，至少包含 human、ai、tool、system 四类来源
- 设计追加式记录和哈希链验证流程
- 设计签名、时间戳、仓库状态和文件路径绑定方式
- 实现最小可用的 create、append、verify 三类操作
- 为格式解析、哈希链验证、签名校验和异常输入添加测试

## 安全边界

- `.bac` 文件应做到篡改可发现，而不是声称绝对不可修改
- 不把密钥、令牌、完整私有提示词或无关隐私写入记录
- 对无法验证的来源信息使用低信任声明，并在格式中显式标记
- 所有验证失败都应返回可解释的原因，避免静默通过

## 里程碑

1. 完成威胁模型和 `.bac` 格式草案
2. 实现本地 CLI 或 library 原型
3. 封装 AI tool 调用接口
4. 添加跨仓库示例和验证文档
5. 评估签名、可信时间戳和远程证明集成
